AutoRes

Small library for automating Java™ resource handling. Use annotations for verification and code generation.

  • Validate resource paths with the compiler
  • Mitigate against stringly typed things
  • Reduce resource file I/O handling by embedding files as byte arrays or strings
  • Replace localization resource bundle messages with typed method signatures
  • Extend with additional generators or validators

This library can affect compile time, jar sizes, and constant pools. Use judiciously.

Documentation assumes prior knowledge of how to package and deploy Java libraries and applications.

Usage

Add the uk.autores:annotations library to the build system.

Browse the central repository for versions and various build system import snippets.

Annotate a type or package with @uk.autores.ClasspathResource and choose a handler:


      import uk.autores.*;

      @ClasspathResource(
        value = {"Poule.txt", "Roses.txt"},
        handler = GenerateStringsFromText.class
      )
      public class PrintRhymes {

        public static void main(String...args)  {
          System.out.println(Poule.text());
          System.out.println(Roses.text());
        }
      }

Dependency Scope

AutoRes is a compile-time only dependency and does not need to be distributed with application code. Apache Maven end consumer projects should use the provided scope. 


    <dependency>
      <groupId>uk.autores</groupId>
      <artifactId>annotations</artifactId>
      <version>${autores.version}</version>
      <scope>provided</scope>
    </dependency>

Libraries that extend the API will need to import using the default compile scope.

See other build system documentation for equivalent scoping.
