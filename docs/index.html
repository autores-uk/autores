AutoRes.uk

Small, open source library for automating embedded Java™ resource file handling. Use annotations for verification and code generation. This is an alternative to manual use of methods like getResourceAsStream.

  • Validate resource paths with the compiler
  • Reduce runtime file I/O handling by embedding files as byte arrays or strings
  • Replace localization resource bundle messages with typed method signatures
  • Extend with additional generators or validators

This library can affect compile time, jar sizes, and constant pools. Use judiciously.

Usage

Browse the central repository for a build system import snippet for the uk.autores:annotations library.

Annotate a type or package with @uk.autores.ResourceFiles, define the resource files, and choose a handler: 


      import uk.autores.*;

      @ResourceFiles(
        value = {"Poule.txt", "Roses.txt"},
        handler = GenerateStringsFromText.class
      )
      public class PrintRhymes {

        public static void main(String...args)  {
          System.out.println(Poule.text());
          System.out.println(Roses.text());
        }
      }

The GenerateStringsFromText handler creates one class per text file that provides the file content as a String.

Other handlers include AssertResourceExists, GenerateByteArraysFromFiles, GenerateConstantsFromProperties, GenerateMessagesFromProperties, and the ability add custom implementations.

Versioning & Compatibility

Versions in the form A.B.C are split into three numbers:

  • A: Java version; minimum version of Java compiler
  • B: the major library version; breaking changes can occur
  • C: the minor library version; breaking changes should not occur

Java 8 is the minimum version. Java 11 is the minimum version with Java Platform Module System support.

🔗

Scopes

AutoRes.uk is a compile-time only dependency and does not need to be distributed with application code.

🔗

Modules

Use requires static uk.autores in module-info.java to depend on this library at compile time only.

🔗

Apache Maven

Apache Maven end consumer projects should use the provided scope. 


      <dependency>
        <groupId>uk.autores</groupId>
        <artifactId>annotations</artifactId>
        <version>${autores.version}</version>
        <scope>provided</scope>
      </dependency>

Java Platform Module System projects should use annotationProcessorPaths to make the compiler aware of the annotation processor service.

Libraries that extend the API should use the default compile scope.
